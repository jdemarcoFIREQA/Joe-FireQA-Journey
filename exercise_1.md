# Test Case Writing Example
These exercises progress in complexity and help you think about different testing angles. The goal is to progress through them daily an become better at markdown, git, and writing test cases. 

Exercise 1: Core Functionality (Happy Path)
- **Goal:** Practice writing the most straightforward, success path for a feature.
- **Prompt:** Write a test case for a user **logging in** to a web application with valid credentials.
- **Focus:** Clarity an adherance to the *Given-When-Then* format.

| Scenario | Given | When | Then |
| -------- | ----- | ---- | ---- |
| *Successful Login* | The user is on the **Login Page** and has registered **Valid Credentials** | The user enters their **valid email** and **password** and clicks the "**Login**" button | The user is **redirected to the Dashboard**(or *Home Page*) an their **Username is displayed** in the header. |
